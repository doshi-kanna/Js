2. Create Tables (Schema)

Below are the SQL statements to create Course, Trainer, and Student tables with foreign keys and checks.

CREATE TABLE COURSE(
  CRSID INT PRIMARY KEY AUTO_INCREMENT,
  CRS_NAME VARCHAR(50),
  CRS_PRICE INT CHECK (CRS_PRICE > 0)
);

CREATE TABLE TRAINER(
  TRID INT PRIMARY KEY AUTO_INCREMENT,
  TRNAME VARCHAR(50),
  TRADDRESS VARCHAR(50)
);

CREATE TABLE STUDENT (
  STID INT PRIMARY KEY AUTO_INCREMENT,
  STDNAME VARCHAR(50),
  STDBATCH INT,
  CRSID INT,
  TRID INT
);

ALTER TABLE STUDENT ADD FOREIGN KEY(CRSID) REFERENCES COURSE(CRSID);
ALTER TABLE STUDENT ADD FOREIGN KEY(TRID) REFERENCES TRAINER(TRID);

5. JOINS — Queries, Simple Definitions & Sample Results

INNER JOIN

SELECT *
FROM STUDENT ST
JOIN COURSE C ON ST.CRSID = C.CRSID;

Definition: Returns only rows that have matching values in both tables.

Multi-table INNER JOIN (Student + Course + Trainer)

SELECT *
FROM STUDENT ST
JOIN COURSE C ON ST.CRSID = C.CRSID
JOIN TRAINER T ON ST.TRID = T.TRID;

Definition: Matches rows across all joined tables where join conditions hold.

LEFT JOIN

SELECT *
FROM STUDENT ST
LEFT JOIN TRAINER T ON ST.TRID = T.TRID
LEFT JOIN COURSE C ON ST.CRSID = C.CRSID;

Definition: Returns all rows from the left table (STUDENT) and matching rows from right tables. If no match, right columns are NULL.

RIGHT JOIN

SELECT *
FROM STUDENT ST
RIGHT JOIN TRAINER T ON ST.TRID = T.TRID
RIGHT JOIN COURSE C ON ST.CRSID = C.CRSID;

Definition: Returns all rows from the right tables (TRAINER / COURSE) and matching rows from left. Useful to see all trainers/courses even without students.

GROUP BY + HAVING Example

SELECT COUNT(ST.STID) AS student_count, T.TRNAME
FROM STUDENT ST
RIGHT JOIN COURSE C ON ST.CRSID = C.CRSID
RIGHT JOIN TRAINER T ON ST.TRID = T.TRID
GROUP BY T.TRNAME
HAVING COUNT(ST.STID) > 1;

Meaning: Count students per trainer, then show trainers with more than 1 student. Sample result: VENKAT — 2

6. Quick Interview Definitions (Short & Clear)

  • INNER JOIN — rows matching in both tables.
  • LEFT JOIN — all rows from left table + matching from right (NULL if none).
  • RIGHT JOIN — all rows from right table + matching from left.
  • FULL OUTER JOIN — (not in MySQL) returns rows when there is a match in either table.
  • GROUP BY — groups rows to aggregate (SUM, COUNT, AVG).
  • HAVING — filters groups (like WHERE but for aggregated groups).
  • FOREIGN KEY — enforces referential integrity between tables.

7. Interview Tips & Short Answers

  1. Explain joins with a one-liner and a tiny example — interviewers like clarity.
  2. Mention referential integrity when asked why use foreign keys.
  3. For optimization mention indexes on join columns (CRSID, TRID).
  4. If asked about NULLs in joins, explain how LEFT/RIGHT produce NULLs when unmatched.
Pro tip: Practice writing queries by hand and explain results aloud — that helps under interview pressure.
