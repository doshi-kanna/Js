  • SELECT – Read data

1.5 TCL – Transaction Control Language

CommandPurpose

ROLLBACKUndo changes

2. SQL Keys

Primary Key

Uniquely identifies records. Cannot be NULL.

Unique Key

Unique values, but can contain NULL.

Foreign Key

Links two tables by referencing a primary key.

Composite Key

Two or more columns uniquely identify a record.

Candidate Key

Columns that can potentially act as Primary Key.

Super Key

Set of attributes that uniquely identifies a record.

5. SQL Joins (Simple Definitions)

INNER JOIN

Returns only matching rows from both tables.

LEFT JOIN

All records from left + matched from right. Unmatched = NULL.

RIGHT JOIN

All records from right + matched from left. Unmatched = NULL.

FULL JOIN

All matched + unmatched from both tables.

CROSS JOIN

Every row from table A is paired with every row from table B.

7. Interview Ready Join Explanation

Inner Join: Matching only
Left Join: All left, matched right
Right Join: All right, matched left
Full Join: All records
Cross Join: All combinations

9. Quick Revision

  • INSERT → Add data
  • UPDATE → Modify data
  • DELETE → Remove data
  • DROP → Delete table or DB
  • PRIMARY KEY → Unique, Not Null
  • FOREIGN KEY → Links tables
  • JOIN → Combines data from tables
